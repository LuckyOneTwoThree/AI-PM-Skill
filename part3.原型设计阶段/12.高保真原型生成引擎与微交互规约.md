# 模块十二：高保真原型生成引擎与微交互规约 (Prototyping & Micro-Interaction)

到了 **「生成原型草图」** 这一步，我们其实面临着当下 AI 工具链的一个巨大现实断层： **目前主流的文本大模型（包括 GPT-4o, Claude 3.5 Sonnet）并不能直接“完美画出”可用的高保真设计图或复杂的 Axure 工程文件。**

很多长 Prompt 在这里会失效，因为你要求大模型同时做“写代码”、“想交互”、“编测试数据”甚至“指导你画 Axure”，大模型的注意力会瞬间崩溃，最终给出一堆无法运行的“半成品代码”或泛泛而谈的“UI 建议”。

为了填补这个断层，让这一模块具备真正的 **工业级“拿来即用”价值**，我将 **「模块十二：高保真原型草图与交互规约」** 重构为 **4个极其垂直、针对不同工具链生态的独立生成路径 + 1个可用性防呆审查**。

---

**👤 AI 角色定位：** 顶级前端架构师 (Frontend Architect) / 资深动效与交互设计师 (UX Motion)

**🎯 核心目标：** 将上游抽象的骨架图，转化为 **可直接在浏览器运行的代码 Demo**，或者 **可供设计师/PM直接照猫画虎的像素级排版参数与交互剧本。**

**📥 前置输入（变量）：** `[模块九的数据字段]`、`[模块十一的页面视觉骨架与组件清单]`、`[你选用的技术栈体系（如 React+Tailwind 或 Ant Design）]`

---

## 🟢 步骤一：数据具象化与“反文案占位符”填充 (Real-Data Mocking)
*操作目的：永远不要在原型里用“Lorem Ipsum（乱数假文）”或“测试标题1”。只有放入带着业务属性的真实数据长度和语境，才能看出版面的真实拉扯感与缺陷。*

- **操作方式：** 先不画图，先让 AI 把“要填进去的字和图”准备好。
- **提示词模板：**
  ```text
  你现在是拥有强业务感的产品经理。在生成任何前端代码或原型之前，我们必须抛弃“测试标题123”这种垃圾占位符。
  
  请基于前述的[数据实体字典]，为即将渲染的 [填写页面名称，如：商品列表页] 生成 5 条极度贴近真实业务场景的 Mock 数据。
  【强制要求】：
  1. 真实长度模拟：包含极长的数据（如超过两行的复杂标题）和极短的数据，暴露出可能导致版面错乱的边缘情况。
  2. 真实情绪植入：如果数据中涉及用户评论、状态提示或警告文案，请用极具人味儿的文字编写，带入用户的焦虑或喜悦情绪。
  3. 结构化输出：请将这 5 条完整的数据集打包为一个格式标准的 JSON 数组，确保其键名（Key）与我们的数据字典字段名完全一致，以便稍后直接注入前端代码或被原型工具读取。
  ```
- **产出物：** 包含极端长度的真实场景 JSON 格式 Mock 数据集。

---

## 🟢 步骤二：单页面极速可视化 Demo 渲染 (Rapid Code-Sandbox Demo)
*操作目的：这一步适用于想要快速在浏览器里“看一眼感觉”或者需要直接给前端提供切图参考的场景。利用 Claude 的 Artifacts 等功能，让 AI 直接吐出可运行的单页应用。*

- **提示词模板：**
  ```text
  请切换为精通 [填写你想要的开源框架，如：React + Tailwind CSS / Vue3 + Element Plus] 的前端专家。
  我们将通过代码快速将刚才的页面骨架和 Mock 数据“具象化”。
  
  请编写一页完整且高度封装的前端代码，要求如下：
  1. 像素级骨架复刻：严格按照上一模块（模块十一）制定的“视觉权重排布”进行布局，优先凸显首屏的 CTA（核心转化按钮）。
  2. 真实数据挂载：将上一步生成的 JSON Mock 数据直接注入，取代写死的文本，并实现一个基础的列表渲染或详情填充。
  3. 基础交互激活：不要写死版！请用简易状态（如 React 的 useState）实现至少 2 个基础交互（例如：点击 Tab 切换内容区、点击“加载更多”出现 Loading 骨架屏加载动画、点击某项弹出抽屉/模态框）。
  4. 一键预览部署：代码必须包含全部必须的 CDN 依赖与内联样式，确保它是一份可以直接粘贴进 CodeSandbox 或单文件浏览器中运行的绝对干净的代码。
  ```
- **产出物：** 单文件、带真实数据、带基础交互、可直接运行预览的极速前端 Demo 代码块。

## 🟢 步骤三：手工原型绘制的像素级指导书 (Pixel-Perfect Spec for Axure/Figma)
*操作目的：如果你的团队不用代码 Demo，而是依赖产品经理手工画 Axure / 墨刀，或者交给 UI 画 Figma。不要让 AI 瞎指挥，让它直接吐出各区域的“长宽比例和间距规范”。*

- **提示词模板：**
  ```text
  如果你不能直接帮我画图，那么请作为资深视觉排版导师，给正在打开 Axure/Figma 准备画原型的我，出具一份毫无歧义的【像素级排版与连线指导书】。
  
  请针对该页面的三大核心区块，输出以下排版参数指令：
  1. 空间与栅格布局指示：不要说“放在左边”，请说“左侧留出20%的流式空间用于导航，右侧80%包裹内容池，各内容卡片之间保持 16px 或 24px 的呼吸间距”。
  2. 核心 CTA 按钮的物理属性：明确定义核心转化按钮的尺寸量级（如高度是否需要 48px/56px 等大尺寸以便于手指触控）、色彩对比度要求、以及其相对屏幕的绝对固钉位置。
  3. Axure/Figma 交互连线剧本：为原型绘制者写出动作剧本。例如：“【动作A】：当用户 Hover 某张卡片时，【触发元件】：该卡片加上外阴影并上浮 4px，同时【状态改变】：右上角淡入显示‘收藏’图标图标组件。”
  ```
- **产出物：** 包含间距/尺寸/栅格约束的排版空间指导书、Axure/Figma 原型动作连线剧本说明。

## 🟢 步骤四：微交互与暗面状态规约 (Micro-Interactions & Dark States)
*操作目的：产品的高级感来源于细节。按钮点下去那零点几秒发生了什么？表单报错是怎么弹出的？必须定义这些被称为“微交互”的魔法瞬间。*

- **提示词模板：**
  ```text
  骨架已经搭好，现在我们需要注入“产品的高级质感”。请化身为极其注重细节的动效与交互微调大师。
  
  请为该页面中最核心的转化交互（如：提交付款、完成点赞、触发大动作检索），补充一份【微交互 (Micro-Interaction) 与暗面状态控制规约】：
  1. 瞬时反馈机制 (Instant Feedback)：当用户按下核心按钮时，在数据返回前的那零点几秒，界面应该发生什么微妙的变化以缓解焦虑？（例如：按钮内文字变为转圈 loading、按钮本身微微下陷变暗、或者触发一次轻微的设备震动提示）。
  2. 阻断性暗面状态 (Dark States)：明确规定在操作的途中，如果发生了什么错误（如：网络突然断开、输入了非法的脏字）。错误提示是采用【强阻断的居中模态框】、还是【轻量级的顶部全局 Toast】、或者是【仅在输入框下方的微小红字 Inline Error】？给出你选择该层级的理由。
  3. 防御性锁死 (Defensive Locking)：为了防止用户在焦躁时产生“双击/狂点”造成脏数据，请明确规定该按钮在点击后的【物理禁用防抖时效】与视觉变灰规则。
  ```
- **产出物：** 核心转化按钮的微动效反馈设计图纸、多级错误提示层级规范、防误触锁死与防抖防重录规则。

## 🛑 原型体验防呆审查（阶段 Checkpoint）
*操作目的：在把这份厚厚的原型规范或者代码 Demo 扔给开发/设计师之前，做一次最后的“傻瓜式通关”检查。*

- **提示词模板（拦截器）：**
  ```text
  作为站在小白用户视角的“防呆审查员”，请审视刚才输出的所有代码逻辑和排版规约：
  
  1. 核心任务迷失排查：如果把一个完全不懂中文、只看图形颜色的用户扔到这个页面，他会不会在 3 秒内被其他花里胡哨的组件吸引走，而找不到真正的下一步按钮？如果是，说明视觉层级出了大问题。
  2. 极端长度击穿测试：回到第一步的 Mock 数据，如果有人的名字长达 50 个字符，结合你第三步的排版规范，整个页面是不是会被撑爆变形？你是否已经强制加入了文字溢出截断（...）或者换行限制机制？
  3. 确认指令：如果页面坚固且能承受各种折腾，微交互充满安全感，请输出“✅ 高保真原型图纸/Demo与交互细节敲定，准许进入[PRD撰写与开发文档化]阶段”。如果发现页面极其脆弱，一碰边缘情况就报错或变形，请直接打回重排。
  ```

---

### 💡 拆解后的优势：

1.  **解决了 AI “不会画图”的痛点：** 既然文字大模型画不了可编辑的 Axure，那就让它（步骤二）写前端代码出 Demo，或者（步骤三）出具高密度的排版参数，完美绕过了模型的短板。
2.  **强制代入真实数据 (Mock Data First)：** 这是一个极为专业的研发习惯（步骤一前置）。摒弃了假文占位，所有的排版和截断规则都是基于真实的、有情绪、有极端长度的数据设计出来的，极大避免了“设计图很丰满，上线后很骨感”的悲剧。
3.  **赋予产品“高级感”：** 专门开辟了一个步骤（步骤四）来打磨微交互、Loading 态和防抖锁死。这是初中级产品经理拉开与高级产品经理差距的核心护城河。你的工作流因此具备了“像素级”的工匠精神。
